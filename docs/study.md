# 学習機能と履歴管理の詳細

学習機能は、単語帳を「覚える」ための核となるロジックです。

## 学習フローの制御 (`public/app.js`)

### `startStudy` 関数 (Study Mode)
1. **周回管理**:
   - `currentWords` (現在の問題セット) と `roundWrongs` (間違えた問題のプール) をメモリ上で管理。
   - 1枚ずつ「フラッシュカード」を表示。
2. **判定ロジック**:
   - ユーザーが「正解（○）」または「不正解（×）」を自己判定。
   - 「不正解」とした場合、その単語を `roundWrongs` に追加。
3. **継続・完了**:
   - 1周終わった時点で `roundWrongs` があれば、それらを次のターンの問題セットとして再スタート。
   - すべて正解するまで繰り返す。

### 学習セッション完了の保存
- 1周（ラウンド）終わるごとに、`/api/study/finish` を叩き、バックエンドに以下の情報を送信：
  - `wordbookId`
  - `wrongWordIds`: そのラウンドで間違えた単語のIDリスト。
  - `testedWordIds`: そのラウンドで今回テストした単語の全IDリスト。

### 中断と再開 (Session Resume)
学習セッションは `localStorage` を使用して進捗が自動保存されます：
- キー: `study_session_${wbId}`
- 保存タイミング: 各単語の回答判定ごと。
- 保存内容: 現在の問題リスト、インデックス、間違えた単語のプール等。
- 再開: 単語帳詳細ページに「続きから再開」ボタンが表示されます。
- クリア: 1周完了時、または「最初から学習」開始時に古いセッション情報は削除されます。

## バックエンドのデータ管理 (`src/routes/study.js`)

### テーブル設計
- **`study_history`**: 学習した証跡を保存。
- **`study_mistakes`**: ユーザーが「まだ覚えていない」とされる単語のリスト。
  - `ON CONFLICT DO NOTHING` により、同じ単語が重複して「苦手リスト」に入らないよう制御。

### 間違い記録の自動クリアロジック
- **上書き保存**: `/api/study/finish` 呼び出し時、`testedWordIds` に含まれる単語の間違い記録を一旦削除し、新たに `wrongWordIds` に含まれる単語のみを登録します。これにより、一度間違えた単語でも正解すれば自動的に「苦手リスト」から外れます。
- **復習モード**: フロントエンドで `isReview=true` で学習を開始。復習モードで個別に「正解」すると、バックエンドの `study_mistakes` レコードがその場で削除（克服）されます。

### 学習履歴の一覧取得 (`GET /api/study/history`)
- 単語帳ごとに「最後にいつ学習したか」と「現在いくつ間違い（苦手）が残っているか」を並行して返します。
- 以下のフィルタパラメータに対応：
  - `uncompleted=true`: 完了ボタン未押下のものを抽出。
  - `mistakes=true`: 間違いが1つ以上残っているものを抽出。

### UIの標準化
- 学習履歴のカード表示にも `.feed-card` クラスを統一して使用し、他のフィードページと一貫したデザイン（余白・ボーダー等）を提供します。

## 単語帳詳細ページでの学習結果表示

### 概要

ログイン中のユーザーが単語帳詳細ページを開くと、単語一覧テーブルに「状況」列が追加され、直近の学習セッションでの正誤が一目でわかります。

### 動作仕様

`renderWordbookDetail` 関数内で `GET /api/study/mistakes/:wordbookId` を呼び出し、間違えた単語の ID を `Set` として保持します。

| 状態 | 表示 |
|------|------|
| 直近の学習で間違えた単語 | 赤い `close` アイコン（ツールチップ: 「直近の学習で間違えた単語」） |
| 直近の学習で正解した単語 | 緑の `check` アイコン（ツールチップ: 「直近の学習で正解した単語」） |
| 未学習（`study_mistakes` が空） | 何も表示しない |
| ゲストユーザー | 「状況」列自体を非表示 |

**正誤の判定ロジック:**
- `mistakeWordIds`（間違い単語IDのSet）が空でない → 学習済みとみなす。
  - 単語が `mistakeWordIds` に含まれる → 間違い (✗)
  - 含まれない → 正解 (✓)
- `mistakeWordIds` が空 → 未学習とみなし、アイコンを表示しない。

### フィルタUIのラベル仕様

フィード・ブックマーク・学習履歴画面のフィルタチップのラベルは、括弧内の補足説明を省いたシンプルな表記にしています。

| フィルタキー | 表示ラベル |
|-------------|-----------|
| `uncompleted` | 未完了 |
| `unstudied` | 未学習 |
| `mistakes` | 間違えあり |
